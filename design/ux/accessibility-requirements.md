# 无障碍需求规格 — Shapeshift Survivor

> **状态**: In Design
> **创建日期**: 2026-05-25
> **作者**: 刘嘉寓 + ux-designer

---

## 概述与哲学

Shapeshift Survivor 的无障碍目标：**让尽可能多的玩家体验"蓄能→爆发"的核心循环，不因视觉、听觉或操作障碍而被排除在外。**

本文件定义了 MVP 阶段必须满足的无障碍最低标准，以及 Vertical Slice 和 Full Vision 阶段的增强目标。每一个交互模式（见 `design/ux/interaction-patterns.md`）都应在实现时对照本标准检查。

**核心原则**：
1. **信息不单一依赖颜色** — 任何通过颜色传达的信息必须有至少一种替代传达方式（形状、位置、文字、动画）
2. **操作可重新映射** — 所有游戏操作必须支持改键；不允许硬编码按键
3. **反馈多渠道** — 重要的游戏事件应同时提供视觉和听觉反馈，且各自独立可用
4. **无运动诱发风险** — 全屏闪烁、屏幕震动、快速视差移动必须可减弱或关闭

---

## 无障碍等级承诺

| 等级 | 阶段 | 定义 |
|------|------|------|
| **Tier 1 — 基础** | MVP | 色盲安全、按键重映射、低 HP 多通道警告、全屏闪光可减弱。这些是发布前必须完成的。 |
| **Tier 2 — 增强** | Vertical Slice | 屏幕震动独立开关、UI 缩放、字幕系统、按住/点击切换选项、高对比度模式 |
| **Tier 3 — 完整** | Full Vision | 屏幕阅读器支持（Godot 4.5+ AccessKit）、完整控制器重映射、游戏速度调节、认知辅助模式 |

## 1. 视觉无障碍

### 1.1 色盲安全

游戏的核心信息绝不应仅依赖颜色传达。以下是对每个关键 UI 元素的色盲安全检查：

| UI 元素 | 当前颜色方案 | 替代传达方式 | 等级 |
|---------|------------|-------------|------|
| HP 条 | 绿(>50%) → 琥珀(30-50%) → 红(<30%) | 条本身宽度变化 + 低 HP 屏幕边缘脉冲（Pattern 7）— 独立于颜色感知 | Tier 1 |
| 形态计量表 | 形态主题色（橙红/紫红） | 填充比例 + 边框脉冲（≥80%时）+ 变身提示文字出现 — 三者独立于颜色 | Tier 1 |
| 持续时间/冷却条 | 形态色/灰色 | 条的方向相反（持续=右→左缩减，冷却=左→右增长）+ 数字倒计时 — 颜色不是唯一区分方式 | Tier 1 |
| 变身提示 | 形态主题色文字 | 脉冲呼吸动画（scale 1.0↔1.1）— 运动本身传达"可按"信息 | Tier 1 |
| 低 HP 警告 | 红色渐变 | 屏幕边缘脉冲（alpha 呼吸）— 即使灰度化也能感知边缘明暗变化 | Tier 1 |
| 升级卡片稀有度 | 边框颜色（绿/蓝/紫/金） | 卡片形状/边框样式 + 稀有度文字标签 — 颜色不是唯一区分方式 | Tier 2 |
| Boss 警告 | 红色闪烁 | "WARNING" 文字 + 屏幕边框同步闪烁 — 文字独立于颜色 | Tier 1 |

**色盲测试场景**（MVP 发布前手动验证）：
1. 将游戏截图转为灰度，确认所有关键信息仍然可读
2. 使用 Deutan 滤镜（红绿色盲，最常见类型）检查 HP 条三种颜色阶段的区分度
3. 使用 Tritan 滤镜（蓝黄色盲）检查龙形态（紫红）与兽形态（橙红）计量表的可区分性

### 1.2 对比度与文字可读性

| 元素 | 最小对比度 | 当前状态 | 等级 |
|------|----------|---------|------|
| 功能文字（HP 数字、倒计时、按键提示） | 4.5:1（WCAG AA） | UI 色板 U1 骨白 `#F0EDE2` 对 U2 暗墨 `#1E1E28` = ~12:1 ✅ | Tier 1 |
| 次要文字（波次、剩余敌人） | 3:1（WCAG AA 大文字） | 需验证 — 灰色文字对游戏背景的对比度取决于当前区域色调 | Tier 1 |
| 按键提示最小字号 | 6px 高（艺术圣经规定） | 6px 像素字体在 1280×720 画布上约等于 12pt — 可读但偏小 | Tier 1 |
| UI 元素与游戏世界的区分 | — | UI 使用独立 8 色调色板（与游戏世界色板分离）— 天然可区分 | Tier 1 |

### 1.3 运动与闪烁

| 风险 | 触发场景 | 缓解措施 | 等级 |
|------|---------|---------|------|
| 全屏闪光 | 变身瞬间（VFX-001）：0.3s 单色闪光 → 0.2s 白帧 → 0.2s 黑帧 | 设置系统 `VFX Intensity: REDUCED` — 跳过白帧和黑帧，仅保留 0.3s 形态色淡入。`OFF` — 完全跳过变身闪光 | Tier 1 |
| 快速闪烁 | 低 HP 临界脉冲：0.5s 周期 alpha 呼吸、Boss 警告：200ms 红闪 ×3 | `REDUCED` — 闪烁周期减慢 50%。`OFF` — 替换为静态边框（低 HP）或静态文字（Boss 警告） | Tier 1 |
| 屏幕震动 | 变身屏幕震动（VFX-003）、Boss 登场震动 | `REDUCED` — 震动幅度减半。`OFF` — 完全禁用 | Tier 2 |
| 大面积粒子运动 | 变身战斗粒子残影、龙息三角粒子流 | `REDUCED` — 粒子数量减至 60%。`OFF` — 仅保留受击白闪和低 HP 警告 | Tier 2 |

**参考**：所有运动缓解措施使用 VFX 系统的三级强度控制（`FULL / REDUCED / OFF`），通过 `VFXSystem.set_intensity(level)` 接口驱动。

### 1.4 UI 缩放

| 需求 | 当前状态 | 等级 |
|------|---------|------|
| UI 元素使用锚点定位，自适应分辨率 | 已实现 — HUD GDD Rule 7 + CanvasLayer `follow_viewport_scale` | Tier 1 |
| 独立 UI 缩放倍率（0.75x – 2.0x） | 未实现 — 需设置系统支持 | Tier 2 |

## 2. 操作无障碍

### 2.1 按键重映射

| 需求 | 当前状态 | 等级 |
|------|---------|------|
| 所有游戏操作支持改键 | Pattern 14 `dynamic-key-label` 已设计为从 InputMap 动态读取 — 架构支持改键。实际改键 UI 需设置系统 | Tier 2 |
| 变身提示文字随改键自动更新 | 已设计 — Pattern 14 每帧从 InputMap 查询最新绑定 | Tier 1 |
| 控制器按钮重映射 | Godot 4.x Input Map 原生支持 — 需设置系统 UI | Tier 3 |

### 2.2 输入替代方案

| 操作 | 默认输入 | 替代方案 | 等级 |
|------|---------|---------|------|
| 移动 | WASD | 方向键、手柄左摇杆/方向键 | Tier 1 |
| 变身 | Space | 手柄 R1/RB — Pattern 5 使用 InputMap 动态读取 | Tier 1 |
| 确认选择 | Enter / Space | 鼠标左键、手柄 A — Pattern 2 | Tier 1 |
| 取消/返回 | Esc | 鼠标右键、手柄 B — Pattern 3 | Tier 1 |
| 菜单导航 | 方向键/WASD | 手柄方向键/左摇杆 — Pattern 1 使用 `ui_*` 动作 | Tier 1 |

### 2.3 按住 vs 点击

| 需求 | 当前状态 | 等级 |
|------|---------|------|
| 变身键：点击触发（非按住） | 已设计 — Pattern 5 响应 `is_action_just("transform_activate")` | Tier 1 |
| 自动攻击：无需持续按键 | 玩家系统 GDD：自动攻击最近的敌人，不需要按键 | Tier 1 |
| 选项：按住替代连点（如菜单导航） | Pattern 1 设计了按住重复（300ms 初始延迟 + 8 次/秒）。可添加设置项切换即时/延迟重复 | Tier 3 |

### 2.4 输入缓冲与防误触

| 需求 | 当前状态 | 等级 |
|------|---------|------|
| 确认后 300ms 输入锁定（防双击跨屏） | Pattern 2 已设计 | Tier 1 |
| 死亡画面 1.4s 输入锁定（防跳过死亡反馈） | Pattern 13 已设计 | Tier 1 |
| 变身失败时视觉反馈（非静默失败） | Pattern 5 已设计 — "shake" 微动画提示失败 | Tier 1 |

## 3. 听觉无障碍

| 需求 | 当前状态 | 等级 |
|------|---------|------|
| 关键游戏事件有视觉等价物 | 变身 = 全屏闪光 + HUD 切换 + 角色模型变化。低 HP = 屏幕边缘脉冲 + HP 数字变化。波次开始 = 文字公告。Boss = 文字警告 — 所有核心事件均有视觉传达 | Tier 1 |
| 独立音量控制（Music / SFX / UI / Voice） | Audio 系统 GDD 预留了四条独立总线 — 需设置系统 UI | Tier 2 |
| 失焦自动静音 | Audio 系统 GDD UI-2：`NOTIFICATION_WM_WINDOW_FOCUS_OUT` 自动静音 | Tier 1 |
| 字幕/视觉音效提示 | 未设计 — Full Vision | Tier 3 |

**听觉无障碍测试**：静音游戏 → 进行一局完整游戏（从探索到死亡）→ 确认所有关键状态变化都能仅通过视觉感知。

## 4. 认知无障碍

### 4.1 信息清晰度

| 需求 | 当前状态 | 等级 |
|------|---------|------|
| UI 仅显示当前状态相关的信息 | Pattern 6 `state-driven-visibility` — 无关元素完全隐藏（alpha=0），减少认知负担 | Tier 1 |
| 信息分层：核心（HP+计量表）> 次要（波次）> 事件（Boss/升级） | HUD GDD Rule 2 三层信息结构 | Tier 1 |
| 倒计时最后 3 秒颜色变化 | Pattern 9 — 琥珀（剩余 3s）→ 红色 + 弹跳（剩余 1s）。时间紧迫的多级提示 | Tier 1 |
| 数值变化使用平滑过渡（非硬跳） | Pattern 8 `bar-fill-lerp` + HUD GDD Rule 4 — 避免数字跳动引发焦虑 | Tier 1 |

### 4.2 一致性

| 需求 | 当前状态 | 等级 |
|------|---------|------|
| 相同操作在不同屏幕使用相同按键 | Pattern 2 `confirm-selection` 和 Pattern 3 `cancel-back` 在所有屏幕统一 | Tier 1 |
| UI 颜色语义一致 | HP=绿/红、不可用=灰色、奖励=金色、确认=琥珀 — HUD GDD Visual/Audio Requirements | Tier 1 |
| 形态颜色主题贯穿所有 UI 元素 | 计量表色 = 倒计时条色 = 变身提示色 — HUD Rule 6 | Tier 1 |

### 4.3 错误预防与恢复

| 需求 | 当前状态 | 等级 |
|------|---------|------|
| 变身不可撤销 — 但失败时给反馈 | Pattern 5：失败时 shake 动画，不变身。玩家知道按键已被接收但条件不满足 | Tier 1 |
| 升级选择不可撤销 — 但需要确认 | Pattern 4：选择后闪光确认 → GSM 状态切换。无"撤销"但确认步骤防止误触 | Tier 1 |
| 死亡后重启需要确认 | Pattern 13：1.4s 延迟 + 需要按键确认 | Tier 1 |

## 5. 平台特定

| 平台 | 特殊考虑 | 等级 |
|------|---------|------|
| **PC (Steam/Epic)** | Steam Input API 支持控制器重映射 — 与游戏内重映射互补 | Tier 3 |
| **Web / Browser** | 浏览器缩放影响 Canvas 尺寸 — `follow_viewport_scale` 已处理。浏览器自动翻译功能可能影响 UI 文字布局 — MVP 仅支持中文 | Tier 1 |

## 6. 验证清单

MVP 发布前逐项确认：

### Tier 1（MVP — 发布前必须通过）

- [ ] 游戏截图转灰度后，所有关键 UI 信息可读
- [ ] Deutan 滤镜（红绿色盲）下 HP 条三阶段可区分
- [ ] 变身提示仅通过动画（非颜色）即可被感知
- [ ] 静音游戏进行一局，所有状态变化可仅通过视觉感知
- [ ] Space 键改绑为 Q 后，变身提示显示 "按 Q 变身"
- [ ] 低 HP 屏幕边缘脉冲在灰度下可感知
- [ ] 全屏闪光减弱后（REDUCED 模式），变身反馈仍清晰
- [ ] 所有交互按钮在 1280×720 下点击区域 ≥24×24px
- [ ] UI 文字与背景对比度 ≥4.5:1（主要信息）/ ≥3:1（次要信息）

### Tier 2（Vertical Slice）

- [ ] 设置界面提供 Music / SFX / UI 三条独立音量滑块
- [ ] 设置界面提供 VFX 强度三级选择（FULL / REDUCED / OFF）
- [ ] 设置界面提供屏幕震动开关
- [ ] 设置界面提供 UI 缩放 0.75x–2.0x
- [ ] 所有操作可在设置界面中重映射
- [ ] 升级卡片稀有度不依赖颜色即可区分

### Tier 3（Full Vision）

- [ ] Godot AccessKit 屏幕阅读器可读取 HP、计量表、波次信息
- [ ] 控制器所有按键可重映射
- [ ] 游戏速度可调节（0.5x / 1.0x / 1.5x）
- [ ] 所有音频提示有对应的视觉字幕/图标

|---|------|--------|------|
| 1 | 像素字体在 6px 高度下对低视力玩家是否可读？艺术圣经规定最小功能文字为 6px。如果需要更大的文字，可能需要打破像素网格约束 | 否 — MVP 先按 6px 实现，收集反馈 | 如果反馈不可读，Tier 2 提供 UI 缩放到 2.0x 作为缓解 |

| 4 | 低 HP 警告的红色渐变对完全色盲（全色盲）玩家是否足够？ | 否 — 脉冲动画（alpha 呼吸）独立于颜色 | 灰度测试可以验证 |
